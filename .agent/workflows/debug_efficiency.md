# 🚀 デバッグ効率化ガイド

## ⚠️ 鉄則：毎回最初からやるな！！

七願は長いフローがある：
```
タイトル → 名前入力 → ステータス配分 → カルマテスト → プロローグ → ジャックイベント → フィールド...
```

**毎回これを全部通すのは時間の無駄！！**

---

## 🎯 デバッグモードを使え！

### 起動方法
1. **タイトル画面の🛠️ Debugボタンをクリック**
2. パスワード入力モーダルが表示される
3. パスワード: `d0154723939` を入力してEnter

### 代替起動方法（ブート画面で）
- ブート画面の隠しパスワード入力欄に直接入力

### できること
- 任意のシーンにジャンプ
- ステータス値の変更
- カルマ値の操作
- イベントフラグの設定

## 📊 効率的なデバッグフロー

### ❌ 悪い例（時間かかりすぎ）
```
1. タイトル画面確認 → スクショ
2. 名前入力 → スクショ
3. ステータス配分 → スクショ
4. カルマテスト → スクショ
5. プロローグ → スクショ
6. ジャックイベント → スクショ  ← やっと本題！
7. 問題発見、修正
8. また1からやり直し...（絶望）
```

### ✅ 良い例（爆速）
```
1. デバッグモードON
2. ジャックイベントに直接ジャンプ
3. 問題確認、修正
4. 再度ジャンプして確認
5. 修正完了！
6. 最後に1回だけフルフローで動作確認
```

## 🎮 テスト用セーブデータ

以下のタイミングのセーブデータを用意しておくと便利：

| セーブ名 | タイミング | 用途 |
|----------|-----------|------|
| `test_title` | タイトル直後 | UI確認 |
| `test_before_karma` | カルマテスト直前 | カルマシステム確認 |
| `test_before_jack` | ジャックイベント直前 | イベント確認 |
| `test_field` | フィールド開始時 | フィールドUI確認 |
| `test_before_battle` | 戦闘直前 | 戦闘システム確認 |

## 📝 スクショのルール

### 撮るべき時
- バグを発見した時（証拠）
- 修正完了の確認時
- フルフローテストの最終確認時

### 撮らなくていい時
- 毎回の通過確認
- 同じ画面の繰り返し確認
- 変更していない箇所
