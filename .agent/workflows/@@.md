# 作業セッション /@@（オートセーブ付き）

このワークフローは **集合知から作業を開始し、完了時に自動でGitに保存** します。

---

## 【作業開始】集合知を読み込む

以下のファイルを読み込んで最新の設定・ルールを把握します：

1. **game_lore.md** - ゲーム設定・ルールの集合知
2. **anti_mogura_debug_guide.md** - デバッグルール＆アンチモグラ叩き ⚠️必読
3. **memory_management_guide.md** - メモリ管理ルール 🧹作業前チェック！
4. **debug_efficiency.md** - デバッグ効率化・ジャンプコマンド集 🛠️
5. **task.md** - 現在のタスク状況

```
view_file: .agent/knowledge/game_lore.md
view_file: .agent/knowledge/anti_mogura_debug_guide.md  ← 毎回必ず！
view_file: .agent/knowledge/memory_management_guide.md  ← 作業前チェック！
view_file: task.md (artifact)
```


---

## 【作業中】ユーザーのリクエストに対応

集合知を参照しながら作業を進めます。

---

## 【作業完了】自動セーブ（Gitコミット＆プッシュ）

作業が完了したら、以下を自動実行します：

// turbo-all

### 1. 集合知の更新（必要に応じて）
作業中に発見した新しい情報やルールがあれば `game_lore.md` に追記。

### 2. タスクリストの更新
`task.md` の完了したタスクに ✅ をつける。

### 3. Gitへ保存
```bash
git add -A
```

```bash
git commit -m "<作業内容の要約（日本語）>"
```

```bash
git push origin main
```

### 4. 完了報告
ユーザーに作業完了とGit保存を報告。

---

## 注意事項
- コミットメッセージは日本語で簡潔に
- 大きな変更がある場合はコミットを分割も検討
